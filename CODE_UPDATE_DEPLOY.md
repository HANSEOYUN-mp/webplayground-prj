# 수정한 코드를 웹사이트에 반영하는 방법

코드를 바꾼 뒤 **배포된 웹사이트(Vercel)** 에 반영하려면,  
**GitHub에 푸시**하면 됩니다. Vercel이 자동으로 다시 배포합니다.

---

## 1. 한 번만 할 일: Supabase에 테이블 만들기

글을 DB에 저장하려면 **Supabase에 테이블**이 있어야 합니다.  
**처음 한 번만** 아래를 실행하세요.

1. **Supabase 대시보드** 접속 → 내 프로젝트 선택
2. 왼쪽 메뉴 **SQL Editor** 클릭
3. **New query** 클릭
4. 프로젝트 폴더의 **`supabase/schema.sql`** 파일을 연다
5. 그 파일 **전체 내용**을 복사해서 SQL Editor에 **붙여넣기**
6. **Run** (또는 Ctrl+Enter) 클릭
7. "Success" 나오면 완료. 이제 글을 저장할 수 있습니다.

이미 테이블을 만들어 둔 적이 있으면 **다시 실행하지 않아도** 됩니다.

---

## 2. 코드 수정 후 웹사이트에 반영하기 (매번)

코드를 수정했을 때, **배포된 사이트**에 반영하는 순서입니다.

### 2-1. 수정한 파일 저장

- Cursor/VS Code에서 **Ctrl+S** 등으로 저장

### 2-2. Git으로 커밋하고 GitHub에 푸시

**방법 A: 터미널 사용**

1. Cursor에서 **터미널** 열기 (Ctrl + `)
2. 프로젝트 폴더가 맞는지 확인 후, 아래 명령어 **순서대로** 실행:

```bash
git add .
git commit -m "글 DB 저장 기능 추가"
git push origin main
```

- `"글 DB 저장 기능 추가"` 대신 **어떤 수정인지** 적어도 됩니다.
- 브랜치 이름이 `main`이 아니면 `git push origin 브랜치이름` 으로 바꾸세요.

**방법 B: 소스트리(SourceTree) 사용**

1. **소스트리** 실행 후 이 프로젝트 저장소 열기
2. **파일 상태**에서 변경된 파일들이 보이면, **전부 선택** (또는 원하는 것만 선택)
3. **스테이징** (Stage) 하기
4. 아래 **커밋 메시지** 입력 (예: `글 DB 저장 기능 추가`)
5. **커밋** 버튼 클릭
6. **푸시(Push)** 버튼 클릭 → **origin / main** (또는 사용 중인 브랜치) 선택 후 푸시

### 2-3. Vercel이 자동으로 다시 배포함

- GitHub에 **푸시가 끝나면** Vercel이 변경 사항을 감지하고 **자동으로 다시 빌드·배포**합니다.
- 1~2분 정도 지나면 배포가 끝납니다.
- **Vercel 대시보드** → 해당 프로젝트 → **Deployments** 탭에서  
  **Building** → **Ready** 로 바뀌는지 확인할 수 있습니다.

### 2-4. 사이트에서 확인

- 배포가 **Ready**가 되면, **Visit** 버튼이나 배포된 주소(예: `https://webplayground-prj.vercel.app`)로 접속해서  
  글 작성 → 새로고침 해도 글이 남는지 확인하면 됩니다.

---

## 요약

| 할 일 | 횟수 | 설명 |
|--------|------|------|
| Supabase에서 `supabase/schema.sql` 실행 | **한 번** | 글 저장용 테이블 생성 |
| 코드 수정 후 `git add` → `commit` → `push` | **수정할 때마다** | GitHub에 올리면 Vercel이 자동 재배포 |

**정리:** 코드 수정 후에는 **커밋 → 푸시**만 하면, Vercel이 알아서 웹사이트를 다시 배포합니다.  
별도로 "배포 버튼"을 누를 필요는 없습니다.
